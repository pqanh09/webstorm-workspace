ovHtmlLegend
directive in module ov-component

Description

Create html chart legends to be used in report.

Usage

as element:
<ov-html-legend
       legend-data="{array}"
       legend-config="{object}">
</ov-html-legend>
as attribute
<ANY ov-html-legend
     legend-data="{array}"
     legend-config="{object}">
   ...
</ANY>

Parameters

ParamTypeDetails
legendDataarray

legend data

legendConfigobject

Legend config.

  • colors string or Array (optional): colors to be used in charts, can be array of color code or d3 colors collection name. If not specified, each object in the data list must carry a color value
  • fixedColors object (optional): A mapper to define fixed color for some data items.
  • mapping object: define map values
    • name string: property name can be used to get legend item name
    • value string: property name can be used to get legend item value

Example

Basic Example: 

 //HTML
 <ov-html-legend legend-config="ctrl.legendConfig" legend-data="ctrl.legendData">
 </ov-html-legend>

//JS
 ctrl.legendConfig = {
    colors: 'colorScale20',
    mapping: {
      name: 'displayName',
      value: 'count'
    },
    fixedColors: {
      Others: '#808080',
      OTHERS: '#808080'
    }
  };
 ctrl.legendData = [
 {
   "displayName": "10.1.4.123",
   "count": 96,
   "color": "#9467bd" // this value gets priority for use rather than legendConfig.colors
 },
 {
   "displayName": "10.1.3.72",
   "count": 281,
   "color": "#ff9896"
 },
 {
   "displayName": "10.1.4.80",
   "count": 554,
   "color": "#aec7e8"
 }];
